VGL Job Help

The following informatoin should help you get started with running jobs on VGL.

Add AWS/NCI Credentials

To run a job in VGL you will need to enter either AWS or NCI credentials in your User Profile. You only need to do this once.
Solutions will not appear until the credentials for at least one cloud service have been provided.

If you have not configured any services you will see this notificaiton in the header panel. Click it and select "Configure Compute Services" to visit the  User Profile screen to add your credentials.
You do not need to enter cloud credentials if you do not intend to run jobs.





Select Data

Data can be selected from the Data Explorer screen.



To do this, search for layers using the faceted search facility on the left and add the required layers with the green "+" button.  Data can then be selected via the "Select Data" button on the map and drawing a rectangular border containing the area of interest. Select  the desired dataset(s) necessary for the job using the dialog that appears and select "Capture Data".




Choose a Solution

Depending on which cloud providers you have suppllied credentials for you will find a number of solutions on the Solutions screen.
Choose the desired solution from the list and click "Select" to add the solution to the cart and make it available to your job.




Create a Job

A job can be created by navigating to the Create Job page. To create a job you must have first selected  the required data from the Data Explorer and at least one solution from Solutions. On the Create Job screen you can:

  • Manage Input Data

    The Input Data panel will show you the selected datasets available to the solution. At least one dataset must be present.



  • Manage Solutions

    The chosen solutions for the job will appear in the Solutions panel. Here you can view the templates for each solution  and make any changes to the template that may be required to further fine-tune the job.



  • Enter Job Details

    This includes information for you to be able to identify your job, as well as compute service parameters to run the job.

    • Job Name
      A name for the job.

    • Job Description (Optional)
      A brief description of the job.

    • Compute Provider
      The compute provider (AWS or NCI) that will run the job. This will be limited to providers you have supplied credentials for,  and some solutions may be restricted to certain providers.

    • Toolbox
      The solution toolbox to use. There will normally only be one option abailable here.

    • Number of CPUs
      The number of CPUs to utilise when computing the job (e.g. 16).

    • Memory (GB)
      The amount of memory (GB) to use when computing the job (e.g. 64).

    • Disk Space (GB)
      The disk space to utilise when computing the job (e.g. 16).

    • Email Notification
      Check this box if you would like to receive an email notification when the job is complete.

    • Set Job Walltime
      Check this box if you would like to apply a walltime to the job.

    • Walltime
      If the Set Job Walltime box has been checked, the walltime (in minutes) can be set here. The job will be cancelled if it exceeds the walltime.



  • Submit Job

    When all the necessary solution parameters and job details have been entered, the "Submit" button can be clicked to send the job to the associated cloud provider  for processing.
    You can view the status of the job or details of the job if it has completed from the Jobs screen.

View Job Results

Jobs that are running or that have been run previously can be viewed on the Jobs screen.
